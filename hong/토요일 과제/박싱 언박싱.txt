박싱은 값 형식을 참조 형식으로 변환 해주는 것을 의미하고
언박싱은 박싱했던 값을 다시 원상태로 복귀 시키는 것을 의미합니다.

박싱
박싱은 값 유형을 유형 object 또는 이 값 유형에 의해 구현된 인터페이스 유형으로 변환하는 프로세스입니다.
인스턴스 내에서 값을 래핑 하고 관리되는 힙에 저장합니다.

int i =123;
object o = i;

코드
int i = 123;
object o = i;
i=456;
출력 o= 123,i= 456를 출력하게 된다.


언박싱
언박싱은 개체에서 값 유형을 추출합니다, 복싱은 암묵적입니다.
언박싱은 명시적입니다.

o =123;
i = (int) o;

코드
int = 123;
object o = i;
int j = (int)o;
출력 o= 123, j =123을 출력하게 된다.

성능
간단할 할당과 관련하여 박싱 및 언박싱은 계산 비용이 많이 드는 프로세스입니다.
값 형식이 박싱되면 새 개체를 할당하고 구성해야 합니다. 덜 박싱에 필요한 캐스트도 계산 비용이 많이 듭니다.
